# 全局写作质量与状态规则

## 1. 状态权威级别

写作系统读取状态时必须区分权威级别：

- 已确稿状态：可作为事实写入正文。
- 待确稿状态：只能作为候选参考，不能写成硬设定。
- 冲突状态：不得直接采用，需等待作者处理。
- 过期状态：仅用于提醒风险，不能作为稳定事实。

## 2. 完稿后回写原则

正文生成或续写完成后，系统可以抽取状态变化，但不能直接写入长期事实。

正确流程：

1. AI 抽取状态变化。
2. 写入状态确稿中心。
3. 作者确认。
4. 写回长期状态表。
5. 后续写作作为已确稿上下文使用。

## 3. 角色状态规则

角色状态变化必须关注：

- 位置。
- 心理状态。
- 关系变化。
- 目标变化。
- 能力或资源变化。
- 伤病、身份、阵营变化。

角色变化进入状态池后，应同步生成角色成长事件。未确稿事件不能作为角色成长事实。

## 4. 冲突与过期规则

当新状态可能推翻旧设定时，不应自动覆盖旧事实。

系统应：

- 标记冲突状态。
- 将受影响的已确稿状态标记为过期。
- 生成影响分析报告。
- 等待作者处理。

## 5. 写作上下文规则

写作上下文必须显式分层：

- 已确稿：主事实层。
- 待确稿：候选参考层。
- 冲突：风险层。
- 过期：复核层。

AI 写作提示词必须说明：待确稿、冲突、过期状态不能被当作稳定事实。

## 6. 锁定章节规则

如果状态修改会影响已锁定章节，系统不能自动应用影响项。

允许：

- 生成影响报告。
- 提醒作者受影响范围。

禁止：

- 自动改写锁定章节。
- 自动覆盖锁定章节中的既定事实。

## 7. 与流程守卫关系

Workflow Guard 判断“当前阶段能不能生成”。

状态确稿中心判断“生成后哪些内容能成为长期事实”。

两个系统互补，不互相替代。
