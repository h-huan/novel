# 小说 Agent 平台终版需求文档

> 保留现有 Tab 工作台，增加 Agent 总控与流程守卫，不做完全重构

版本：V1.1 | 日期：2026-07-03 | 适用项目：h-huan/novel

# 一、终版结论

**当前项目不需要推倒重构。正确方向是保留现有 Tab 功能页，把 Agent 做成上层流程编排、阶段守卫、智能检查和自动化任务执行层。Tab 是工作台，Agent 是导演，状态库是记忆，Workflow Guard 是规则。**

> 最终产品形态：  
> Tab 工作台 + Agent 总控层 + Workflow Guard + Canon State Vault + Prompt Chain + RAG

- 现有 Tab 页面继续保留，用于精确编辑、查看、管理结构化资产。

- Agent 不替代 Tab，而是作为项目 Dashboard、页面侧栏、后台流程的智能编排层。

- 长篇创建应支持两个主要来源：从灵感开始；从想法开始并经 AI + 手动逐渐完善后再创建。

- 短篇和长篇不是创建方式，而是作品类型；灵感、想法、导入、空白才是创建来源。

- 短篇严格执行三步骤：题材 → 大纲 → 正文；长篇严格执行两百万字级长篇创作全流程。

- 下一阶段重点是 UI 层级调整、流程守卫、项目元数据、想法孵化器、状态确稿中心，而不是重写已有模块。

# 二、纠正一个关键概念：作品类型 ≠ 创建来源

当前容易重复的写法是把“短篇、长篇、从灵感创建、从想法创建、手动创建、导入创建”放在同一级。这个设计会让用户困惑：到底是在选作品类型，还是在选创建方式。

| **维度** | **正确选项**                                        | **说明**                                               |
|----------|-----------------------------------------------------|--------------------------------------------------------|
| 作品类型 | 短篇 / 长篇                                         | 决定后续写作流程、数据结构、质检标准和平台适配方式。   |
| 创建来源 | 灵感 / 想法 / 导入 / 空白                           | 决定项目从哪里开始，以及前置引导要问哪些问题。         |
| 目标平台 | 知乎盐选 / 番茄 / 起点 / 抖音故事 / 小红书 / 自定义 | 决定语言节奏、钩子密度、反转节奏、章节长度和发布格式。 |
| 创作流程 | 短篇三步骤 / 长篇全流程                             | 决定系统允许用户进入哪些阶段，不能混用。               |

推荐把新建作品流程改成两步或三步，而不是一次性堆很多按钮。

> 第一步：你想从哪里开始？  
> - 从灵感开始  
> - 从想法开始  
> - 导入已有资料  
> - 空白创建  
>   
> 第二步：你要创作什么类型？  
> - 短篇  
> - 长篇  
>   
> 第三步：目标平台是什么？  
> - 知乎盐选 / 番茄 / 起点 / 抖音故事 / 小红书 / 自定义

# 三、长篇创建的两个正确入口

## 3.1 从灵感开始创建长篇

适合用户还没有明确故事，只是从热点、灵感库、题材卡、脑洞推荐中挑选方向。该入口不是直接创建项目，而是先判断灵感是否适合长篇化。

> 灵感发现 → 灵感卡片 → AI 判断长篇潜力 → 生成 3 个长篇方向 → 用户选择平台与字数 → AI 生成基础设定 → 用户确认 → 创建长篇项目 → 进入世界观 / 角色 / 大纲流程

- 复用现有灵感发现、项目创建、长篇大纲、角色、世界观等模块。

- 新增“长篇潜力评估”：是否能支撑 50 万字、100 万字、200 万字。

- 灵感确认前不要生成完整项目，避免垃圾项目过多。

## 3.2 从想法开始创建长篇

这是长篇更常见、更重要的入口。用户可能只有一句模糊想法，需要 AI 追问、补全、打磨，并允许人工逐步完善。只有当想法成熟后，才创建正式项目。

> 一句话想法 → AI 追问 → 用户补充 → AI 提供强化方向 → 用户选择或混合 → 生成高概念 → 判断长短篇适配 → 生成基础设定 → 用户确认 → 创建项目

| **字段**                                   | **说明**                                   |
|--------------------------------------------|--------------------------------------------|
| rawIdea                                    | 用户最初输入的一句话想法。                 |
| aiQuestions                                | AI 根据想法生成的追问。                    |
| userAnswers                                | 用户对追问的补充。                         |
| genre / platform / targetLength            | 题材、平台、目标体量。                     |
| protagonistSeed / conflictSeed / worldSeed | 主角、冲突、世界观种子。                   |
| maturityScore                              | 想法成熟度评分。                           |
| status                                     | draft / refining / confirmed / converted。 |
| convertedProjectId                         | 确认后转成正式项目的 ID。                  |

# 四、短篇也可以从灵感或想法开始，但流程必须不同

短篇同样可以从灵感、想法、导入、空白开始，但进入项目后必须执行短篇三步骤。不能把短篇做成长篇流程，也不能把长篇压缩成短篇三步骤。

| **作品类型** | **允许的创建来源**        | **进入后的强制流程**                                                                                  |
|--------------|---------------------------|-------------------------------------------------------------------------------------------------------|
| 短篇         | 灵感 / 想法 / 导入 / 空白 | 题材生成 → 第一人称完整大纲 → 天龙 8 步正文 → 终稿质检 → 标题简介 → 平台适配。                        |
| 长篇         | 灵感 / 想法 / 导入 / 空白 | 基础设定 → 世界观 → 人物 → 组织地图 → 全书大纲 → 分卷大纲 → 章节规划 → 日更写作 → 状态归档 → 周复盘。 |

# 六、Tab 是否要改成 Agent：不改，做组合

Tab 和 Agent 是互补关系。Tab 负责结构化资产管理，Agent 负责流程推进、缺漏检查、自动调用、状态归档。全部改成对话式 Agent，会丢失小说创作所需的精确资产管理能力。

| **形态**       | **职责**                                                             | **保留/新增**          |
|----------------|----------------------------------------------------------------------|------------------------|
| Tab 工作台     | 编辑角色、世界观、大纲、章节、伏笔、地图、组织、时间线、素材、精修。 | 保留并优化。           |
| 项目总控 Agent | 判断当前阶段、提醒缺失资产、提示下一步、展示风险。                   | 新增在 Dashboard。     |
| 页面侧栏 Agent | 在当前 Tab 内做检查、补全、建议和局部生成。                          | 逐步新增。             |
| 后台流程 Agent | 章节写完后提取状态、生成待确稿、检查连续性、更新伏笔状态。           | 基于现有状态管理增强。 |
| Workflow Guard | 控制短篇/长篇不能越级生成，避免流程乱跳。                            | 新增服务层。           |

> Tab 是工作台  
> Agent 是导演  
> 状态库是记忆  
> Workflow Guard 是规则  
> RAG 是资料检索  
> Chain 是执行流水线

# 九、Workflow Guard：先做轻量规则，不做复杂 Agent 框架

第一阶段最该做的是流程守卫，而不是大规模 Agent 重构。Workflow Guard 只负责判断当前项目处于什么阶段、能否执行下一步、缺少哪些资产、是否需要用户确认。

| **项目类型** | **流程规则**                                                                                                                                   |
|--------------|------------------------------------------------------------------------------------------------------------------------------------------------|
| 短篇         | 没有题材不能生成大纲；没有大纲不能写正文；正文完成后进入终稿质检、标题简介、平台适配。                                                         |
| 长篇         | 没有基础设定不能生成世界观；没有世界观不建议生成人物；没有核心人物不能生成全书大纲；没有分卷大纲不能批量生成章节；章节写完必须状态提取与确稿。 |

> WorkflowGuardService 核心能力：  
> 1. getCurrentStage(projectId)  
> 2. getAllowedActions(projectId)  
> 3. getMissingAssets(projectId)  
> 4. assertCanGenerateOutline(projectId)  
> 5. assertCanGenerateChapter(projectId)  
> 6. requireStateConfirmation(projectId)  
> 7. buildNextStepSuggestion(projectId)

# 十、长篇总流程终版

> 从灵感 / 想法 / 导入 / 空白开始  
> → 基础设定  
> → 详细世界观  
> → 核心人物  
> → 组织与地图  
> → 全书大纲  
> → 分卷详细大纲  
> → 章节规划  
> → 每日章节写作  
> → 状态提取  
> → 作者确稿  
> → 下一章上下文构建  
> → 周复盘  
> → 后期整体修改  
> → 发布维护

- 长篇不是一次性生成 400 章，而是分层规划、分段执行、持续归档。

- 世界观、人物、组织、地图、时间线、伏笔必须先于大规模正文生成。

- 每章生成后必须进入状态提取和作者确稿，否则后续章节不能把变化当成事实。

- 周复盘用于检查节奏、伏笔、角色成长、读者反馈和下周写作计划。

# 十一、短篇总流程终版

> 从灵感 / 想法 / 导入 / 空白开始  
> → 阶段一：题材生成  
> → 阶段二：第一人称完整大纲  
> → 阶段三：天龙 8 步正文生成  
> → 终稿质检  
> → 标题简介  
> → 平台改写  
> → 发布导出

- 短篇的核心是强钩子、第一人称代入、递进反转、伏笔回收和平台适配。

- 短篇不要走长篇的复杂世界观流程。

- 短篇也需要状态记录，但状态粒度可以小于长篇。

# 十三、验收标准

- 新建作品入口不再出现“短篇、长篇、从灵感、从想法、手动、导入”同级重复。

- 用户能清晰完成：选择创建来源 → 选择作品类型 → 选择目标平台 → 进入对应流程。

- 长篇支持从灵感开始和从想法开始两种核心路径。

- 从想法开始时，AI 能追问并允许用户手动完善，确认成熟后再创建项目。

- 短篇项目不能跳过题材和大纲直接写正文。

- 长篇项目不能跳过世界观、人物、总纲、分卷规划直接批量写正文。

- Dashboard 能显示当前阶段、下一步建议、待确稿事项和风险。

- 正文生成上下文只读取已确稿事实，不读取待确稿状态。

- 现有 Tab 页面可继续使用，且不因新增 Agent 总控而失效。
