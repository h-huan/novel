# 短篇/长篇页面简化规则规范（子衿交付）

> 来源：writing-workflow-expert（子衿）
> 用途：指导 UI 设计和前端实现

## 一、切换逻辑

根据 `ProjectType` 判断：
- `short_story` → 短篇简化模式
- `long_novel` → 长篇完整模式
- `script` → 剧本模式

切换时机：项目创建时选择，项目设置中可修改（需提示数据隐藏风险）。

---

## 二、短篇模式

### 世界观页面（WorldPage）
**保留**：
- 单卡片模式（无 Tab）
- 时代背景（下拉）
- 核心地点（1-3个，标签输入）
- 基本社会规则（文本域，200字）
- 故事前提（一句话）
- 特殊设定（折叠式可选）

**隐藏**：地理志、历史年表、势力组织、力量体系、地图绘制、关系图

### 人物页面（CharacterPage）
**保留**：
- 最多5个人物卡片
- 简化属性：姓名、身份、核心目标、性格标签（3-5个）、故事功能（主角/反派/助攻等）

**隐藏**：关系网络图、弧光跟踪、多人物对比、深层心理分析

### 大纲页面（OutlinePage）
**保留**：
- 线性章节列表（无卷结构，建议5-20章）
- 天龙8步法检查表（默认视图）：目标、诱因、行动、阻碍、误判、反转、代价、钩子
- 章节列表视图（核心事件+冲突强度+情感基调）
- 节奏曲线图

**隐藏**：卷结构、多线叙事、伏笔管理系统、情感曲线

---

## 三、长篇模式

### 世界观页面
完整6 Tab：地理志、历史年表、社会规则、力量体系（条件显示）、势力组织、特殊设定

### 人物页面
完整功能：分组管理、完整档案（6大区域）、关系网络图、弧光跟踪、出场章节统计

### 大纲页面
完整功能：卷管理（3-5卷）、章节管理、多线叙事、伏笔管理系统、反转计划表、情感曲线图、人物出场跟踪

---

## 四、组件复用策略

| 模块 | 短篇组件 | 长篇组件 |
|------|----------|----------|
| 世界观 | `WorldSimpleCard` | `WorldTabView` |
| 人物 | `CharacterSimpleCard` | `CharacterDetailPanel` |
| 大纲 | `OutlineSimpleView` | `OutlineVolumeView` |

同一路由，根据 `project.type` 动态渲染不同组件。
