# 第四阶段：长短篇流程硬约束实施说明

版本：V1.0  
日期：2026-07-07

---

## 一、第四阶段目标

本阶段把第三阶段的 Workflow Guard 从“提示 + 判断 + 轻量阻断”升级为关键生成动作前的强校验。

核心目标：

1. 页面仍可自由进入和查看。
2. 关键 AI 生成动作必须先经过 Workflow Guard。
3. 短篇遵守：题材 -> 大纲 -> 正文。
4. 长篇遵守：想法 / 灵感 -> 世界观 -> 人物 -> 总纲 -> 分卷 -> 章节规划 -> 正文。
5. 后端做最终校验，前端只做提前提醒和交互优化。

---

## 二、短篇硬流程规则

短篇类型：`short_story`

阶段：

```text
topic -> outline -> writing
```

规则：

- `topic` 阶段允许生成题材和生成大纲，但不能生成正文、续写正文或导出终稿。
- `outline` 阶段允许生成和编辑大纲；没有大纲时不能进入正文生成。
- `writing` 阶段允许生成正文、续写、精修和质量检查。
- 正文生成前必须满足：当前阶段为 `writing`，且已有大纲。

---

## 三、长篇硬流程规则

长篇类型：`long_novel`

阶段：

```text
idea_or_inspiration
-> world_setting
-> character
-> outline
-> volume
-> chapter
-> writing
```

规则：

- 未完成世界观和主角设定前，不能生成总纲或正文。
- 未完成总纲前，不能生成分卷。
- 未完成分卷前，不能生成章节规划。
- 未完成章节规划前，不能生成正文或日更正文。
- 正文生成前必须满足：当前阶段为 `writing`，且已有章节规划。

---

## 四、后端强校验接入点

新增或扩展的 Workflow Guard 方法：

- `assertActionAllowed(projectId, action)`
- `assertCanGenerateOutline(projectId)`
- `assertCanGenerateVolume(projectId)`
- `assertCanGenerateChapterPlan(projectId)`
- `assertCanGenerateBody(projectId)`
- `assertCanContinueBody(projectId)`

已接入的 Chain 入口：

- `POST /chain/generate`
- `POST /chain/continue`
- `POST /chain/long-write`
- `POST /chain/long-outline-generate`
- `POST /chain/templates/execute/:id`，大纲类模板且输入含 `projectId` 时校验
- `POST /chain/stream-generate`

流程阻断返回 HTTP 400，并包含：

- `error = WorkflowGuardBlocked`
- `missingAssets`
- `currentStage`
- `recommendedNextAction`

---

## 五、前端阻断策略

前端不阻止用户进入页面，只在关键生成动作前提前校验。

写作面板：

- `AI生成` 前检查 `generate_body`
- `AI续写` 前检查 `continue_body`
- `长篇生成` 前检查 `generate_body`
- 不通过时显示 `WorkflowBlockedNotice`，不调用 Chain 接口

大纲页：

- `AI生成完整大纲` 和 `重新生成` 前检查 `generate_outline`
- 不通过时显示缺失资产和下一步建议

流程助手：

- 当 `canProceed = true` 且存在 `recommendedNextStage` 时显示“进入下一阶段”按钮
- 点击后调用 `POST /projects/:projectId/workflow-guard/advance`
- 后端继续防越级

---

## 六、与第三阶段 Workflow Guard 的关系

第三阶段提供了流程状态、缺失资产、允许动作、不建议动作和阶段推进接口。

第四阶段复用第三阶段能力，并增加：

1. 关键动作强校验。
2. 后端 assert 方法。
3. ChainController 生成入口接入。
4. 前端生成按钮前置检查。
5. 可复用阻断提示组件。

Workflow Guard 仍是唯一流程判断来源，规则不散落到各页面中。
