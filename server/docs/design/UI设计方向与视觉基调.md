## 一、设计系统 (Design System)

### 1.1 命名：墨砚 (MoYan)

「墨」取书写作之意，「砚」承载墨色、沉淀思考。设计系统代号 MoYan，所有 Token 以 `my-` 为前缀。

### 1.2 色彩体系

#### 设计理念

写作是长时间沉浸式工作，色彩方案遵循三个核心原则：
- **护眼优先**：避免纯白/纯黑，底色带暖调，降低蓝光刺激
- **层级克制**：主色仅用于关键操作与状态指示，大面积区域保持中性
- **语义清晰**：成功/警告/错误/信息 四种语义色区分明确


#### 关键动效规范

| 场景 | 类型 | 时长 | 说明 |
|------|------|------|------|
| **按钮hover** | 背景色渐变 + 微阴影 | 150ms | 所有交互按钮 |
| **侧边栏折叠** | 宽度过渡 | 250ms | 280px ↔ 48px(图标模式) |
| **右侧面板滑入** | 从右滑入 + 淡入 | 250ms | 大纲/角色/伏笔面板 |
| **模态框** | 缩放淡入 (scale 0.96→1) | 200ms | 确认/设置弹窗 |
| **AI流式输出** | 逐字淡入 | 持续 | Typewriter效果, 50ms/字 |
| **AI思考状态** | 三点脉冲 + 光晕旋转 | 循环 | 生成等待时 |
| **骨架屏** | shimmer 从左到右 | 1500ms | 内容加载中 |
| **差异对比高亮** | 颜色从亮渐变为正常 | 800ms | 切换差异时 |
| **字数统计更新** | 数字递增动画 | 600ms | 编辑器底栏 |
| **Toast通知** | 从右侧滑入 + 3s后淡出 | 250ms in / 300ms out | 保存/操作结果 |
| **拖拽排序** | 跟随指针 + placeholder收缩 | 实时 | 大纲树/章节排序 |

## 五、交互模式规范

### 5.1 侧边栏分屏交互

```
┌─────────────────────────────────────────────────────────┐
│  触发方式:                                               │
│  · 编辑器内 Ctrl+Shift+R → 打开/关闭右侧面板             │
│  · 点击大纲/角色/伏笔导航 → 右侧面板切换对应Tab           │
│  · 编辑器内选中角色名 → 右侧自动切换角色面板              │
│                                                         │
│  面板行为:                                               │
│  · 默认宽度 320px，可拖拽左边缘调整 (280~420px)          │
│  · 面板打开时编辑器自动缩小，不遮挡内容                   │
│  · 面板关闭动画：从右滑出 250ms                          │
│  · 面板内滚动独立于主区域                                │
│                                                         │
│  面板联动:                                               │
│  · 角色面板 → 点击角色名 → 高亮编辑器中该角色出场段落     │
│  · 大纲面板 → 点击章节 → 编辑器跳转到该章节              │
│  · 伏笔面板 → 点击伏笔 → 高亮埋设/回收位置的段落         │
│  · 质检面板 → 点击问题 → 编辑器跳转+高亮问题行           │
└─────────────────────────────────────────────────────────┘
```

```
│  · 精修完成 → 底部「查看对比」按钮                       │
│  · 版本历史 → 选择两个版本 → 「对比所选」               │

│  交互规则:                                               │
```

## 七、暗黑模式策略

### 7.1 实现方案

采用Tailwind CSS的 `dark:` 前缀 + CSS变量双轨制：

```css
/* 在 :root 和 .dark 中定义同一组CSS变量 */
:root {
  --color-bg-page: #faf9f7;
  --color-bg-card: #f3f1ed;
  --color-text-primary: #2d2820;
  /* ... 所有Token */
}

.dark {
  --color-bg-page: #1a1a18;
  --color-bg-card: #242320;
  --color-text-primary: #ddd7ce;
  /* ... 对应暗色值 */
}
```

```tsx
// 组件中使用
<div className="bg-[var(--color-bg-page)] text-[var(--color-text-primary)]">
```

### 7.2 切换方式

```
1. 自动跟随系统
   · 默认策略，首次安装跟随系统设置
   · 使用 matchMedia('(prefers-color-scheme: dark)') 监听系统变化

2. 手动切换
   · 快捷键: Ctrl+`
   · 设置页面: 外观 > [亮色] [暗色] [跟随系统]
   · 偏好持久化到 localStorage

3. 切换动效
   · 过渡时间 300ms
   · 不是瞬间切换，而是从点击位置扩散的圆形渐变
   · 使用 CSS View Transitions API (如果可用)
```

### 7.3 暗色模式特殊适配

## 八、响应式表达

### 8.1 窗口尺寸策略

虽然目标是桌面应用，但窗口大小可变（分屏使用、不同分辨率显示器）。

| 断点 | 最小宽度 | 布局调整 |
|------|----------|----------|
| **Compact** | 960px | 不支持。显示「请放大窗口」提示 |
| **Standard** | 1280px (最小) | 左侧栏折叠(48px) + 主区域 + 右侧面板关闭 |
| **Comfortable** | 1440px | 左侧栏展开(280px) + 主区域 + 右侧面板折叠(48px) |
| **Optimal** | 1600px+ | 左侧栏展开(280px) + 主区域 + 右侧面板展开(320px) |

### 8.2 各断点适配方案

#### Standard (1280~1439px)

```

· 左侧导航强制折叠为图标模式
· 右侧面板默认关闭（需手动打开，以Sheet形式滑入）
· 编辑器占据最大可用宽度
· 适合小屏笔记本或分屏使用场景
```


### 8.3 窗口大小变化处理

```
· 窗口resize时平滑过渡（无闪烁、无布局跳动）
· 过渡时间: 250ms (仅面板宽度)
· 编辑器宽度使用CSS clamp() 限制:
   编辑器最小宽度: 500px (低于此值进入溢出滚动)
   编辑器最大宽度: 1100px (超过此值居中显示，两侧留白)

· 低于960px: 显示覆盖层「请将窗口调整至960px以上」
· 恢复至960px+: 自动移除覆盖层，恢复之前的布局状态
```

### 8.4 编辑器内部响应式

Monaco编辑器内部通过以下方式适配不同宽度：

```typescript
// 根据编辑器容器宽度自动调整
const editorOptions = {
  // 字数换行
  wordWrap: containerWidth < 700 ? 'on' : 'off',
  // 行号显示
  lineNumbers: containerWidth < 500 ? 'off' : 'on',
  // 小地图
  minimap: { enabled: containerWidth > 800 },
  // 字体大小
```
