### 1.2 短篇创作流程（三步骤法）

```
┌─────────────────────────────────────────────────────────┐
│              短篇：三步骤创作流程（目标3000-10000字）     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  步骤一：题材生成 ────────────────────────────────────  │
│    输入：素材(真实经历/新闻/关键词/片段) + 平台风格      │
│    输出：3~5个题材（含钩子/身份/场景/冲突/反转/评分）    │
│    Chain：素材解析→风格分析→脑洞发散→筛选→报告          │
│                                                         │
│  步骤二：大纲生成 ────────────────────────────────────  │
│    输入：选定的题材 + 平台风格约束                       │
│    输出：核心设定 + 人物关系 + 章节结构 + 递进反转表     │
│          + 伏笔回收表（≥3次反转 + ≥8个伏笔）             │
│    Chain：设定→人物→章节→反转表→伏笔表→细化→校验→报告   │
│                                                         │
│  步骤三：正文生成 ────────────────────────────────────  │
│    输入：大纲 + 角色卡 + 世界观约束                      │
│    输出：逐章正文（天龙8步法驱动）                       │
│    Chain：目标→诱因→行动→阻碍→误判→反转→代价→钩子       │
│                                                         │
│  外挂模块：开头强化 / 反转强化 / 平台改写 / 标题简介     │
│           终稿质检 / 版权检测 / 敏感词检测                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 二、功能优先级矩阵 (P0/P1/P2)

### 2.1 优先级定义

| 级别 | 定义 | 判断标准 | 不做的后果 |
|------|------|---------|-----------|
| **P0** | 必须有 | 没有它整个产品无法闭环，核心价值主张无法实现 | 产品不可用 |
| **P1** | 应该有 | 显著提升用户体验和创作效率，但不是阻断性的 | 产品能用但竞争力不足 |
| **P2** | 可以有 | 锦上添花，差异化增值功能 | 不影响核心创作流程 |

### 2.2 全功能优先级矩阵

#### A. 灵感发现 (Inspiration Discovery)

| # | 功能 | 优先级 | 理由 | 依赖 |
|---|------|--------|------|------|

| **RAG记忆持久化** | 基础级别 | 三层架构（P0，长篇记忆累积量巨大） | 🟡 共用增强 |
| **素材库** | 可选 | 推荐（风格资产化保障长篇一致性） | 🟡 共用 |
| **质检维度** | 7/10维度（热血/伏笔密度可降权） | 全10维度（热血/伏笔密度为核心） | 🟡 共用增强 |
| **导出** | 单文件即可 | 分卷导出+完整项目包 | 🟡 共用增强 |
| **写作模式** | 全自动/半自动/自由均可 | 推荐半自动（逐步确认，降低长周期偏差） | 🟡 共用 |

### 4.2 短篇专属需求

```
短篇核心 = 快速 + 高度反转 + 强钩子

1. 题材生成是唯一入口——必须上来就给3~5个选择
2. 大纲一次性生成，不需要分卷
3. 反转表≥3次反转是硬指标
4. 每章结尾钩子是生死线（短篇读者随时可以划走）
5. 不需要24维状态（角色少，状态变化简单）
6. 不需要锁定机制（一次写完，不存在"前面锁了后面没法改"的问题）
7. 外挂模块更重要：开头强化/反转强化/平台改写
8. 全自动模式接受度更高（短篇容错率高）
```

### 4.3 长篇专属需求

```
长篇核心 = 一致性 + 长期记忆 + 节奏控制

1. 世界观是地基——65条约束必须一开始就建好
2. 24维状态引擎是生命线——没有它，50章后人设必然崩坏
3. 8种章节功能分配是节奏控制器——没有它，读者必然弃书
4. 伏笔生命周期管理是品质保障——42个伏笔在手，不能漏任何一个
5. 章节衔接是叙事流畅度的底线——章与章不能有断裂感
6. 锁定机制是安全网——锁定=已确认，未锁定=可调整
7. 冲突优先级是自动修正系统——锁定正文是最高真理
8. Author's Note是方向盘——作者必须能随时控制方向
9. RAG持久化三层架构——长篇记忆量巨大，丢失=灾难
10. 导入引擎是迁移入口——已有100章存量不能从头写
```

### 4.4 工作流分叉逻辑

```
用户创建项目
  │
  ├─ 选择「短篇」
  │    ├ 入口：灵感面板 → 题材生成(A)
  │    ├ 跳过：世界观建设(C)——简化为3~5条约束
  │    ├ 跳过：24维状态(D2)——不需要
  │    ├ 简化：大纲(E)——2层即可
  │    ├ 简化：伏笔(F)——简易表
  │    ├ 简化：锁定(G5)——整篇锁定
  │    ├ 强化：外挂(开头/反转/改写)
  │    └ 流程：生成→精修→质检→导出 (3~5天)
  │
  └─ 选择「长篇」
       ├ 入口：项目创建 → 世界观建设 → 角色建设 → 素材库
       ├ 必须：65条约束(C2) + 24维状态(D2)
       ├ 必须：4层大纲(E1) + Goal弧线(E2) + 章节功能(E3)
       ├ 必须：伏笔生命周期(F1) + 反转分层表(F2)
       ├ 必须：锁定机制(G5) + 冲突优先级(R) + Author's Note(G4)
       ├ 必须：章节衔接(G3) + 3连续检查(G10)
       ├ 必须：RAG三层持久化(P)
       └ 流程：设定→大纲→逐卷逐章正文→精修→质检→导出 (数月)
```

### 5.2 SWOT分析

#### 本平台优势 (Strengths)
1. **方法论体系化**：短故事三步骤 + 天龙8步法 + 长篇叙事结构，三位一体，是竞品不具备的系统性
2. **反转管理**：递进反转表是行业首创，直接解决"反转不合理/反转不惊人"的核心痛点
3. **章节衔接**：紧/跳/平行三种衔接模式，长篇连贯性的独家保障
4. **去AI味+版权检测**：平台反AI的关键能力，两个竞品均未覆盖
5. **导入AI拆解**：从已有作品自动提取世界观/角色/伏笔，迁移成本远低于竞品
6. **冲突优先级体系**：显式的冲突金字塔，长篇一致性维护的自动化方案

#### 本平台劣势 (Weaknesses)
1. **尚未发布**：游蜂和蛙趣已有多年的用户积累和迭代
2. **素材库从零起步**：蛙趣有1392条素材和拆书资产积累
3. **精修模板数量**：蛙趣22套 vs 我们初始数量未知
4. **生态缺失**：蛙趣有素材市场/模板市场，我们v1.0没有
5. **社区/教程**：从零建设

#### 机会 (Opportunities)
1. **AI写作监管趋严**：平台对AI检测越来越严格，去AI味是刚需
2. **版权意识觉醒**：作者越来越担心侵权和被侵权
3. **多平台分发**：一篇内容多平台变现，平台改写能力是差异点
4. **长篇连续创作痛点**：竞品长篇一致性做得不够好，我们有24维+RAG+锁定三位一体
5. **本地+隐私**：BYOK模式满足对数据敏感的头部作者

#### 威胁 (Threats)
1. **游蜂有桌面端先发优势**：用户迁移成本
2. **蛙趣的短剧/分镜生态**：打入影视化链条
3. **AI基座模型能力提升**：如果GPT-5/Claude-4自带极强的写作一致性，我们的RAG差异化会削弱
4. **免费替代方案**：ChatGPT/Claude直接Prompt写作

### 5.3 关键差距弥补策略

| 差距 | 策略 | 优先级 | 预计时间 |
|------|------|--------|---------|
| 素材库从零积累 | Phase 1内置参考数据+支持用户导入积累 | P1 | 持续 |
| 精修模板数量不足 | 模板系统设计为可扩展框架，v1.0内置核心8套 | P0 | Phase 6 |

### 5.4 核心竞争力一句话总结

> **游蜂强在大纲架构 + 状态追踪 | 蛙趣强在素材生态 + 多形态输出**
> **本平台 = 游蜂的大纲方法论 + 蛙趣的RAG记忆 + 独家的反转/衔接/去AI味/版权检测**
> **差异化核心：让AI写出来的东西「不像是AI写的」+「长篇不崩」**
