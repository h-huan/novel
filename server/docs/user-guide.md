# AI 写作平台 · 用户指南

---

## 1. 安装指南

### 系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| 操作系统 | Windows 10 / macOS 12 / Ubuntu 20.04 | Windows 11 / macOS 14 / Ubuntu 22.04 |
| CPU | 4 核 2.0GHz | 8 核 3.0GHz+ |
| 内存 | 8GB | 16GB+ |
| 磁盘 | 10GB 可用空间 | 50GB+ SSD |
| Node.js | v18.0+ | v20.0+ |
| ChromaDB | 可选（用于 RAG 增强） | 推荐安装 |

### 下载安装

1. 从项目仓库克隆或下载最新版本：
   ```bash
   git clone <仓库地址>
   cd novel-ai-platform
   ```

2. 安装后端依赖：
   ```bash
   cd server
   npm install
   ```

3. 安装前端依赖：
   ```bash
   cd ../desktop
   npm install
   ```

### 首次启动

1. **启动后端服务**：
   ```bash
   cd server
   npm run start:dev
   ```
   服务默认运行在 `http://localhost:3100`。

2. **启动桌面应用**：
   ```bash
   cd desktop
   npm run dev
   ```

3. 在浏览器中打开 `http://localhost:3000`（开发模式）或使用 Electron 桌面客户端。

### 后端服务启动

后端基于 NestJS 框架，提供以下启动方式：

- **开发模式**（热重载）：`npm run start:dev`
- **调试模式**：`npm run start:debug`
- **生产模式**：先 `npm run build`，再 `npm run start:prod`

## 2. 快速入门

### 5 分钟快速教程

#### 第一步：创建项目

1. 启动应用后，点击"新建项目"按钮
2. 输入项目名称（如"我的第一部小说"）
3. 选择类型（玄幻/都市/历史等）
4. 点击"确认创建"

#### 第二步：设置世界观和角色

1. 进入项目后，点击"世界观"选项卡
2. 填写基本的时代背景、地理设定
3. 切换到"角色"选项卡
4. 点击"添加角色"，填写角色名和性格特征
5. 至少创建 1-2 个主要角色

#### 第三步：开始写作

1. 点击"大纲"选项卡，规划基本章节结构
2. 切换到"写作"选项卡
3. 点击"新建章节"，输入章节标题
4. 在编辑器中开始写作
5. 支持自动保存（每 30 秒）

#### 第四步：导出作品

1. 写作完成后，点击"导出"按钮
2. 选择导出格式（Markdown / TXT / HTML）
3. 点击"导出"
4. 文件将自动下载到本地

## 3. 核心功能

### 项目管理

- **创建项目**：点击首页"新建项目"，填写名称和类型
- **项目列表**：首页展示所有项目，支持搜索和排序
- **项目信息**：点击项目进入详情页，查看统计数据和进度

### 世界观编辑

- **设定管理**：添加时代背景、地理环境、社会制度等信息
- **约束管理**：设置 65 条时代约束规则，确保内容一致性
- **时代检测**：自动检测文本中的时代词汇是否一致

### 角色系统

- **角色卡**：为每个角色创建详细档案，包括姓名、性格、背景等
- **24 维状态**：跟踪角色的 24 个维度状态（健康、情绪、人际关系等）
- **五维图谱**：可视化展示角色的五个核心维度

### 大纲规划

- **多层级大纲**：支持卷→章→节的层级结构
- **Goal 弧线**：为每章设置目标弧线（积累→爆发→收束）
- **章节功能**：标记每章的功能（铺陈/推进/高潮/转折）

### 伏笔管理

- **生命周期**：伏笔从"埋设"→"激活"→"回收"→"完成"的全流程管理
- **回收预警**：超过指定章节未回收的伏笔会发出警告
- **状态统计**：查看项目中所有伏笔的状态分布

### 正文写作

- **天龙 8 步**：支持全自动/半自动/手动三种写作模式
- **F1 模式**：自由创作，无 AI 干预
- **F2 模式**：半辅助，AI 提供建议
- **F3 模式**：全自动，AI 完成整章生成
- **沉浸视图**：全屏写作模式，排除干扰

### 精修质检

- **Describe 逐句**：逐句精修，提供不同风格的改写方案
- **降 AI**：检测并降低 AI 生成痕迹
- **十维度评分**：从开头钩子、热血感、代入感等维度综合评分
- **错别字检查**：自动检测错别字和语法问题

### 导入导出

- **导入格式**：支持 TXT / Markdown / DOCX / EPUB
- **导出格式**：支持 Markdown / TXT / HTML / EPUB
- **发布预览**：查看不同平台上的排版效果

## 4. AI 功能

### 单章节 AI 写作

- AI 写作面板默认使用“全自动”模式；仍可按需要切换为半自动或手动模式。
- 在“生成目标”下拉框中选择要写的章节。每次生成只会写入这一章，不会自动连带生成后续章节。
- 生成时系统优先读取该章节绑定的详细大纲，并同时注入同一项目已确认的角色、世界观、地点、时间线、伏笔和前文状态。
- 初稿生成后会保存到所选章节；状态、摘要、RAG、伏笔与时间线的后续同步仍需按页面提示完成或重试。存在 `needs_resync` 或连续性冲突时不得锁定章节。
- 若章节尚未关联详细大纲，系统会阻止生成并提示先在大纲页关联；不会用其他章节大纲、仅标题或通用资料替代。

### Prompt Chain 编排

Prompt Chain（提示词链）是一个可视化的写作工作流编辑器：

- **可视化编辑器**：拖拽式节点编排，支持串行/并行/条件分支
- **预设模板**：内置多种写作流程模板（灵感生成→大纲→正文→质检）
- **自定义节点**：支持添加 LLM 调用、规则检查、数据转换等节点
- **执行测试**：可在编辑器中直接执行并查看每个节点的输出

### 多模型路由

- **BYOK（Bring Your Own Key）**：支持接入自己的 API Key
- **Key 管理**：在设置中添加和管理多个 API Key
- **日常模型**：在“系统设置 → 模型切换”为当前写作模式选择日常模型。没有在“指定任务模型”中单独配置的全部 AI 调用都会使用它。
- **指定任务模型**：灵感、大纲、写作、优化、质检等任务可单独指定模型；指定任务优先于日常模型。保存后立即参与实际路由，不会只改变界面显示。
- **模型切换**：支持 Claude / GPT-4o / DeepSeek 等多种模型；未配置日常模型时才沿用原有场景路由。

### 章节动态目标字数

- 每个可写章节的目标字数必须来自它关联的详细大纲，且必须是 3200–4000 字之间的具体整数。
- 章节列表、分卷列表和编辑器读取同一份大纲目标；不会为了消除提示而拿项目总字数或固定默认值替代。
- 如果详细大纲本身没有有效目标，正文生成会停止，需先完成该章节的剧情任务、场景和篇幅规划。

### 敏感词检测与替换

- **内置词库**：5 大类敏感词（政治/暴力/色情/歧视/违法）
- **自动替换**：支持 replace（替换）/ remove（移除）/ warn（警告）三种策略
- **平台适配**：针对番茄/起点/晋江等平台预设不同严格级别
- **形近字识别**：自动检测拼音/谐音/形近字变体

### 版权检测

- **标题检测**：与 200+ 知名作品标题进行相似度对比
- **角色名检测**：检测角色名是否与已有作品中的角色名相似
- **内容指纹**：自动比对内容片段，识别潜在侵权风险
- **黑名单/白名单**：用户可自定义版权过滤规则

## 6. FAQ

### Q1: 运行后报错"ChromaDB 连接失败"怎么办？

A: 这是正常现象。系统会自动降级使用内存向量存储，所有功能不受影响。如需使用 ChromaDB 增强检索功能，请单独安装并启动 ChromaDB 服务：
```bash
docker run -p 8000:8000 chromadb/chroma
```

### Q2: AI 生成质量不好如何调整？

A: 可从以下几个方面优化：
1. 在 Prompt Chain 编辑器中调整提示词（更详细的描述）
2. 切换不同的底层模型（Claude / GPT-4o / DeepSeek）
3. 在设置中调整生成参数（温度/最大长度等）
4. 提供更详细的大纲和角色设定

### Q3: 如何导入已有小说？

A: 支持以下方式：
1. **文本导入**：复制粘贴纯文本内容
2. **文件导入**：支持 TXT / Markdown / DOCX / EPUB 格式
3. **项目包导入**：支持 .novel 格式的完整项目包
导入后系统会自动识别章节结构和角色信息。

### Q4: 如何导出为不同格式？

A: 在项目页面点击"导出"按钮，选择目标格式：
- **Markdown**：适合继续编辑
- **TXT**：纯文本格式
- **HTML**：适合网页发布
- **EPUB**：适合电子阅读器
- **.novel 项目包**：完整项目备份（含角色/世界观/大纲）

### Q5: 锁定章节后怎么修改？

A: 锁定章节是为了防止误修改。如需修改：
1. 在章节列表中找到锁定章节
2. 右键点击选择"解锁"
3. 修改完成后可重新锁定

### Q6: 如何管理自定义词典？

A: 自定义词典用于配置敏感词和专业术语：
1. 进入"设置" → "词典管理"
2. 可添加/编辑/删除自定义词汇
3. 支持批量导入（JSON 格式）
4. 敏感词词典支持设置策略（替换/移除/警告）

### Q7: ChromaDB 连接失败怎么办？

A: 请检查：
1. ChromaDB 服务是否已启动（默认端口 8000）
2. 网络连接是否正常
3. 防火墙是否阻止了连接
4. 系统会自动降级使用内存存储，不影响基本功能

### Q8: 性能太慢如何优化？

A: 可尝试以下优化：
1. 升级硬件（特别是内存和 CPU）
2. 关闭不需要的 AI 功能（如质检、版权检测）
3. 使用轻量级模型（如 DeepSeek 替代 GPT-4o）
4. 减少同时打开的项目数量
5. 定期清理历史版本数据

### Q9: 如何恢复误删除的章节？

A: 系统自动保存版本历史：
1. 进入章节详情页
2. 点击"版本历史"选项卡
3. 选择要恢复的版本
4. 点击"恢复此版本"
系统保留最近 50 个版本快照。

### Q10: 多人在线协作是否支持？

A: 当前版本为单用户桌面应用，不支持多人在线协作。但可通过以下方式实现间接协作：
1. 使用 .novel 项目包导出/导入共享
2. 版本历史可用于追踪不同人的修改
3. 未来版本将支持 WebSocket 实时协作

### Q11: AI 痕迹指数太高怎么办？

A: 使用"降 AI"功能：
1. 选中需要处理的文本
2. 点击"降 AI"按钮
3. 选择处理强度（轻度/中度/重度）
4. 查看 diff 对比，确认修改
建议在最终质检前执行降 AI 处理。

### Q12: 如何查看项目的写作进度？

A: 在项目详情页可查看：
1. 总章节数 / 已完成数
2. 总字数 / 目标字数
3. 角色数量
4. 伏笔状态分布
5. 每日写作统计
